# 資産運用シミュレーションゲーム 〜お弁当マスターへの道〜

## 1. プロジェクト概要

本作は、ブログ記事「中学生からわかる資産運用 〜お弁当メタファーで学ぶ投資と自衛〜」の内容を体感できるブラウザベースのシミュレーションゲームです。
プレイヤーは1ターン（1年）ごとに手持ちのコイン（資産）を5つの投資先に配分し、20ターン（20年）後の最終資産額と称号（エンディング）を目指します。

**技術スタック**: React + TypeScript + Vite (Antigravity IDEによる自律生成を想定)

## 2. ゲームの基本ループと初期設定

* **初期資産（スタート時の所持金）**: プレイヤーおよび3人のライバルCPUは、全員共通で **「100コイン」** を所持した状態からスタートする。（※100枚＝100%として、配分比率を直感的にわかりやすくするため）

ゲームは全20ターンで構成され、毎ターン以下のフェーズを繰り返します。

1. **回収フェーズ（初回スキップ）**: 前ターンの市場変動を反映したコインが、プレイヤーの手元（Wallet）にドドドッと音を立てて戻ってくる。

2. **配分（アロケーション）フェーズ**: プレイヤーは手元の全コインを、中央の5つのフィールドに自由に割り振る。

3. **市場変動フェーズ（競馬風演出）**: ターン終了ボタンを押すと、すぐに結果を出すのではなく、5つの資産が「今年の変動率」に向かってグラフ上で競争するように伸びていくアニメーション（バーチャートレース風）を表示し、着地点で結果を確定させる。

## 3. 操作感と演出（インタラクション・触覚・音響）

「自分で決めて投資している感覚」を最大化するため、以下のリッチなUI/UXを実装すること。

* **ベット操作の定義**:

  * **マウスホイール**: フィールド上で「上スクロール」でコインを追加、「下スクロール」でコインを手元に戻す。

  * **長押し**: 「左クリック長押し」で連続追加、「右クリック長押し」で連続回収。

* **触覚フィードバック (Haptics)**:

  * モバイル端末向けに `navigator.vibrate()` API を使用。コインが動くごとに微弱な振動（例: `10ms`）、大暴落発生時には激しい画面振動（CSSのスクリーンシェイク）と共に長い振動パターンを再生する。

* **動的サウンド (Web Audio API)**:

  * コインを連続でベットし続けると、コンボのように「チャリン」という効果音のピッチ（音階 / `playbackRate`）が徐々に上がっていく演出を入れる。

## 4. UI・画面構成（16:9 横長画面想定）

※開発初期の画像素材（キャラ、アイコン）は、まずは「絵文字（Emoji）」や「シンプルなSVG図形」をプレースホルダーとして使用してロジックを完成させ、後日差し替え可能な構造（コンポーネント分離）にしておくこと。

* **InfoPanel（左側）**:

  * 現在のターン数（Turn X / 20）

  * 発生中のイベント通知（ポップな文字で）

  * 過去から現在までの5資産の価格推移を示す折れ線グラフ（Market Chart）

* **AllocationField（中央）**:

  * 5つの資産（現金、国内債券、外国債券、国内株式、外国株式）のベットエリア（円形または弁当箱風）。

  * 各エリアに現在の配置コイン数を表示。

* **PlayerWallet（中央下部）**:

  * 未配分の手持ちコイン数と「決定（ターン終了）」ボタン。

* **RivalLeaderboard（右側）**:

  * プレイヤーと3人のライバルの現在順位・総資産額をリアルタイム表示。順位変動でUIが上下に入れ替わる。

* **UtilityMenu（ヘッダー等）**:

  * 「リタイア（最初からやり直す）」ボタンを設置。誤操作防止のため、必ず確認用モーダルを挟むこと。

## 5. ライバルCPUの仕様

* **ギャンブラー・ケン**: 毎ターン「外国株式」に全額ベット。好景気で爆発するが、暴落で一気に沈む。

* **ビビりのカメオ**: 毎ターン「預金（現金）」に全額ベット。暴落は無傷だが、インフレイベントで徐々に資産が削られる。

* **パニックのロウ**: 基本はバランス型（株50:債券50）だが、「大暴落イベント」発生の次ターンにパニックを起こし、預金に全額避難（狼狽売り）する。

## 6. パラメータとイベント仕様（AI調整要件）

1ターンの変動率は `基準リターン + ランダム値( -リスク幅 〜 +リスク幅 )` で計算。
※AIへの指示：**王道の分散投資（5等分リバランス）が勝率・安定性において最適な結果となるよう、モンテカルロシミュレーション等で内部的にバランスを微調整して実装すること。**

| **資産クラス** | **基準リターン** | **リスク（変動幅）** | **備考** | 
| ----- | ----- | ----- | ----- | 
| **預金（水）** | + 0% | ± 0% | 絶対に減らないがインフレに弱い | 
| **国内債券** | + 2% | ± 3% | ディフェンスの要 | 
| **外国債券** | + 4% | ± 7% | ミドルリスク | 
| **国内株式** | + 6% | ± 15% | 成長エンジン | 
| **外国株式** | + 8% | ± 20% | ハイリスク・ハイリターン | 

**【ランダムイベント（上書き補正）】**

* **大暴落（〇〇ショック）**: 国内・外国株式の変動率を強制的に `-30% 〜 -50%` にする（全20ターン中1〜2回発生）。

* **インフレの波**: 預金の変動率を強制的に `-5%` にする（定期的に発生）。

* **好景気ブーム**: 株式の変動率に `+10%` のボーナスを加算。

## 7. 終了条件と称号（エンディング）

20ターン終了時、最終的な資産額とプレイスタイルに基づきリザルト画面を表示する。

* **ドリームライフ（資産額による分岐）**: 「宇宙旅行」「南の島生活」「現状維持」などのビジュアル。

* **プレイスタイル称号**:

  * 【最高ランク】三ツ星の幕の内職人 / 伝説の弁当マスター（バランス良く分散・リバランスを維持した場合）

  * 【極端プレイ】胃もたれ焼肉大王（株全振り）、水飲み修行僧（現金全振り）、弁当ひっくり返し魔（頻繁な極端な配分変更）

## 8. 【重要】AIエージェント向け 開発上の制約事項（アンチパターン回避）

* **DOM描画の最適化（フリーズ対策）**: プレイヤーの資産が数千コインに膨れ上がった際、コイン1枚につき1つのDOM要素を描画するとブラウザがクラッシュします。コインの移動エフェクトは「代表的な数個のパーティクル」と「数字のカウントアップ処理」のみで表現し、パフォーマンスを最優先してください。

* **無限ループの防止**: Reactの `useEffect` フックを使用する際、依存配列の管理不足による無限レンダリングが発生しないよう、状態（State）の更新フローは単方向かつシンプルに保つこと。

* **安全な状態管理**: 市場の変動率計算やライバルAIの行動決定など、ビジネスロジックはコンポーネントの外部（純粋関数やカスタムフック）に分離し、テスト可能な構造にすること。
